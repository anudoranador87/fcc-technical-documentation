## 📖 About this project

This is the **Technical Documentation Page** project for the [freeCodeCamp Responsive Web Design](https://www.freecodecamp.org/learn/2022/responsive-web-design/) certification.  
Instead of writing about a generic JavaScript framework, I decided to document **my own learning process**: the mindset, the tools, the daily log, the projects (like EquiShift) and the way I work.

The page is fully responsive, includes a fixed navbar on the left (desktop) and adapts to mobile with a media query.  
The visual style is inspired by my [Dev Log 365](https://anudoranador87.github.io/Mi-Camino-Web-365/) — warm paper tones, clean typography, and a professional yet personal feel.

---

## 📂 Sections

1. **Filosofía del aprendizaje** – Consistency, error as part of the path, my “Fénix” mindset.
2. **Dev Log 365** – The structure of my daily journal: STAR, debug log, mental model, next target.
3. **Stack técnico y progreso** – Honest skill levels, certifications and real projects.
4. **EquiShift Málaga** – The real‑world problem behind my flagship project: fair shift rotation for hotel teams.
5. **Herramientas y método** – My workflow, debugging technique, tools (VS Code, Git, GitHub Pages).

Each section contains paragraphs, code examples, and lists – exactly as required by the freeCodeCamp user stories.
